## 设计作用

Jade 是整合包的**世界探索 QoL 核心**。作为方块/实体信息提示器（Hwyla 分支），它在玩家准星指向目标时显示关键信息，大幅降低玩家面对大量模组方块时的认知负担。

支持的设计支柱：
- **UI 一致性**：Jade 的提示框是全游戏覆盖最广的 UI 元素之一，其主题、位置和动画必须与整合包整体视觉风格协调。当前使用 `jade:dark` 主题以匹配暗色 HUD。
- **性能优先**：Jade 本身设计目标为轻量级，但配置不当（如显示过多实体信息）可能导致 FPS 下降，特别是在密集实体区域。

Jade 与 JEI 形成互补：JEI 回答"我能合成什么"，Jade 回答"我面前这是什么"。

## 玩家可见行为

### 可发现性
- 默认启用，玩家进入世界后无需任何操作即可看到准星提示。
- 按 `NumPad1`（默认绑定）可切换显示模式（按住/切换/隐藏）。

### 交互
- 准星指向方块时，显示方块名称、所属模组、工具类型、硬度、当前方块状态（如红石信号强度、容器内容预览）。
- 准星指向实体时，显示实体名称、生命值、所属模组、部分实体特有信息（如村民职业、动物繁殖冷却）。
- 指向容器方块（箱子、熔炉等）时，Jade 可预览内部前几个物品槽的内容（需配置开启）。

### 进度与奖励
- Jade 本身不涉及进度系统，但它是玩家识别新方块、判断工具选择、发现隐藏信息（如作物生长阶段）的关键辅助工具。

### 摩擦与失败状态
- 提示框遮挡画面中心，在战斗或精细建筑时可能干扰视线。
- 若服务端未安装 Jade，客户端仍可使用，但部分信息（如容器内容、红石信号）可能不准确或缺失。
- 某些模组方块若未正确实现 `IWailaPlugin` 接口，Jade 可能只显示基础名称而无详细信息。

## 集成关系

### 依赖
- **NeoForge** `>=21.0.143`（required, BOTH）
- **KubeJS Additions**（间接）：KubeJS Additions 将 Jade 列为可选客户端依赖（`>=15.10.5+neoforge`），意味着 KubeJS 脚本有能力为 Jade 注册自定义提示信息。未来魔改方块时，应通过此机制暴露设计意图信息。

### 冲突
- 与原版 WAILA、Hwyla 不能共存。当前直接选用 Jade 作为唯一方案。
- 与某些添加自定义 HUD 的模组（如迷你地图、状态栏增强）可能在屏幕空间上竞争。

### 配方与标签
- Jade 本身不添加配方或物品。KubeJS 若添加自定义方块，需确保其 `BlockBehaviour` 正确设置描述信息，以便 Jade 读取。

### 任务与世界生成
- 无直接关联。

### UI
- Jade 提示框渲染在准星附近，默认锚点为屏幕顶部中央偏下（`overlayAnchorX: 0.5, overlayAnchorY: 0.0`）。
- 所有 GUI 界面中自动隐藏（`hideFromGUIs: true`），避免与菜单、背包界面重叠。
- Boss 血条出现时自动下移（`bossBarOverlapMode: PUSH_DOWN`），防止信息被遮挡。

### 资源
- 支持通过数据包/资源包添加 Jade 插件，为特定方块/实体注册自定义提示提供者。

### 加载顺序
- Jade 在 `BOTH` 侧加载。服务端负责同步部分方块状态数据到客户端。

## 配置意图

以下配置选择反映了整合包的设计意图，**非全部配置项的逐条记录**。

### `config/jade/jade.json`（主配置）

#### `general` 段
- **`displayMode: TOGGLE`**：玩家按一次快捷键即可永久切换显示/隐藏，比 `HOLD` 模式更适合长时间建筑或探索。减少按键疲劳。
- **`displayBlocks: true` / `displayEntities: true`**：同时显示方块和实体信息。整合包定位包含大量新方块和实体，两者都需要识别。
- **`fluidMode: ANY`**：指向流体时显示信息。对水下探索或流体处理模组（如 Create）至关重要。
- **`perspectiveMode: CAMERA`**：提示框跟随相机视角，而非玩家实体位置。在第三人称视角下体验更自然。
- **`hideFromTabList: true` / `hideFromGUIs: true`**：在 Tab 玩家列表和所有 GUI 界面中隐藏提示框，避免干扰社交和物品管理。
- **`builtinCamouflage: true`**：自动隐藏伪装方块的真实身份（如某些模组的隐藏门）。整合包定位为生存冒险，保留探索神秘感。
- **`itemModNameTooltip: true`**：在物品提示中显示所属模组名称，帮助玩家快速定位物品来源。

#### `overlay` 段
- **`alpha: 0.7`**：提示框背景 70% 不透明度。足够看清文字，同时不完全遮挡后方世界。平衡可读性与沉浸感。
- **`activeTheme: jade:dark`**：使用暗色主题，与 Minecraft 默认 HUD、JEI 暗色风格一致。后续若整合包推出自定义主题，需更新此字段。
- **`overlayPosX: 0.5, overlayPosY: 1.0`**：提示框水平居中、垂直贴顶。这是玩家视线最容易捕捉的位置，同时避免与准星重叠。
- **`overlayScale: 1.0`**：默认缩放比例。若未来适配 4K 分辨率，可能需要提高到 1.5。
- **`animation: true`**：启用出现/消失动画，提升 UI 一致性体验。

#### `formatting` 段
- **`itemModNameStyle: { italic: true, color: blue }`**：模组名称以蓝色斜体显示，与物品名称形成视觉层级区分，同时不喧宾夺主。

### `config/jade/plugins.json`
- 控制各模组插件的启用状态和详细程度。当前为默认配置，所有插件启用。未来若某些模组提供的信息过于冗余（如显示全部 NBT），可在此禁用特定插件。

### `config/jade/sort-order.json`
- 控制提示框中各信息行的显示顺序。当前为默认顺序。后续若需要突出显示特定信息（如能量存储优先于模组名称），可调整此文件。

### `config/jade/hide-blocks.json` / `hide-entities.json`
- 当前为空或仅包含少量条目。预留用于隐藏技术性方块（如 KubeJS 占位方块）或调试实体，避免玩家看到无意义信息。

## 兼容性与性能

### 客户端/服务端范围
- 标记为 `BOTH`。服务端必须安装以正确同步方块状态数据（如容器内容、红石信号）。
- 客户端可单独安装并使用基础功能，但信息完整度下降。

### 存档影响
- 无直接存档数据。配置为客户端本地存储。

### 已知不兼容
- 与某些全覆盖 HUD 模组（如显示坐标、指南针的模组）可能在屏幕顶部竞争空间。
- 部分模组的自定义渲染管线可能与 Jade 的 overlay 渲染冲突，导致提示框闪烁或位置偏移。

### 资源成本
- 每帧需进行准星射线检测、目标查询和文字渲染。在实体密集区域（如动物农场、刷怪塔）FPS 可能下降 5-10%。
- 容器预览功能需要查询服务端容器数据，可能产生额外网络流量。

### 缓解措施
- 默认 `disappearingDelay: 0.0`：准星移开后立即消失，避免持续占用渲染资源。
- 限制服务端同步频率：Jade 内部已有节流机制，但超高频更新场景（如红石时钟）仍可能产生网络负担。
- 建议玩家在刷怪塔等场景临时隐藏 Jade（`NumPad1` 切换）。

## 风险与开放问题

1. **服务端-客户端版本不一致**：若服务端与客户端 Jade 版本不匹配（如服务端未更新），容器预览等功能可能失效或导致控制台报错。需在更新流程中强制同步版本。
2. **信息过载风险**：随着模组数量增加，Jade 提示框可能变得过于冗长（一行接一行）。需要在 `plugins.json` 中持续审查并禁用低价值信息。
3. **与自定义 HUD 模组的空间竞争**：若未来添加迷你地图或状态栏模组，需统一规划 HUD 布局，避免 Jade 提示框被遮挡或遮挡其他元素。
