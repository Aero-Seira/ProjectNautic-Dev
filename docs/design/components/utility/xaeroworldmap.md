## 设计作用

Xaero's World Map 是 ProjectNautic 整合包的客户端全屏世界地图模组，自动记录玩家探索过的区域，提供缩放、平移、路径点管理与已探索地形查看。它与 Xaero's Minimap 形成互补：小地图负责实时导航，世界地图负责宏观规划。该模组支撑整合包**QoL 工具**与**Vanilla+ 起步**的设计意图，在不破坏原版探索乐趣的前提下，为玩家提供可靠的区域信息参考。

Xaero's World Map 为 CLIENT 侧模组，可与 Xaero's Minimap 集成以增强体验。

## 玩家可见行为

- **可发现性**：玩家按默认按键（通常是 `M`）打开全屏世界地图，查看已探索区域。
- **交互**：
  - 缩放与平移地图
  - 添加、编辑与管理路径点
  - 查看已探索区块与未探索区域的边界
  - 若同时安装 Xaero's Minimap，可共享路径点与地图数据
- **进度/奖励**：无直接进度关联。间接帮助玩家规划探索路线与建筑布局。
- **摩擦**：无额外摩擦。地图数据随探索自动记录。
- **失败状态**：地图缓存损坏可能导致部分区域显示异常，通常可通过删除缓存重建。

## 配置意图

`config/xaero/world-map/` 目录下包含以下默认生成文件：

- `client.cfg`
- `common.cfg`
- `profiles/default.cfg`
- `server_profiles/default.cfg`

这些文件当前均为**生成默认值，未做 deliberate 覆盖**。整合包选择保留 Xaero 的默认世界地图体验，与小地图策略保持一致：默认配置已足够支持 Vanilla+ 探索需求，未来根据运营反馈再决定是否统一调整（如路径点颜色、地图更新频率、多人共享策略）。

特别说明：Xaero's World Map 支持多维度地图独立缓存，当前 `common.cfg` 为生成默认值，未对特定维度做排除或特殊处理。未来若添加自定义维度，需验证世界地图是否正确识别并分别缓存。

## 兼容性与性能

- **客户端/服务端范围**：标记为 BOTH，但核心功能在 CLIENT 侧。服务端安装可支持部分数据同步。
- **存档影响**：无直接存档影响。地图缓存存储在客户端 `xaero/` 目录下，换设备不保留。
- **已知不兼容**：与 JourneyMap、Dynmap 等地图模组不建议同时安装。
- **资源成本**：世界地图缓存随探索范围增长，长时间运行后可能占用较多内存与磁盘空间；首次打开大地图时可能有短暂加载。
- **缓解措施**：默认配置已启用合理的缓存管理；若出现性能问题，可建议玩家定期清理 `xaero/world-map/` 缓存。

## 风险与开放问题

1. **地图缓存膨胀**：长期运营服务器中，玩家客户端的世界地图缓存可能非常庞大；需考虑是否提供缓存管理指南。
2. **多人地图数据一致性**：服务端与客户端地图数据同步机制需在实际多人环境中验证。
3. **与未来领地/声明模组的兼容性**：已预留 `openpartiesandclaims` 可选集成，未来若添加领地模组需验证兼容性。
