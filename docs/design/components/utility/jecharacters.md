## 设计作用

Just Enough Characters 让玩家在 JEI 搜索栏中可以使用拼音首字母或全拼搜索中文物品名，而不必切换输入法输入中文。它支撑整合包**中文玩家体验与信息可达性**的设计意图，显著降低在中国玩家社区中检索大量模组物品的操作门槛。

该模组为 CLIENT ONLY。

## 玩家可见行为

- **可发现性**：玩家在 JEI 搜索栏输入拼音（如 `tie` 或 `tk`）时，即可匹配到中文名包含对应拼音的物品。
- **交互**：无需切换输入法，直接以英文键盘输入拼音进行过滤；支持全拼与多种常见模糊音。
- **进度/奖励**：无直接关联。
- **摩擦/失败**：若拼音库未覆盖某些生僻译名或自定义 KubeJS 物品名，可能无法命中。

## 集成关系

- **依赖**：无显式依赖声明；功能上深度依赖 JEI 的搜索系统。`mods.toml` 未列出 mandatory dependency。
- **冲突**：与 REI / EMI 的拼音搜索方案若同时安装可能产生冲突；当前包只使用 JEI。
- **协作**：与 **JEI** 协作，扩展其搜索栏的拼音匹配能力；与 **I18nUpdateMod** 协作，后者提供中文译名，使拼音搜索有内容可匹配。
- **配方/标签**：无。
- **任务**：无。
- **世界生成**：无。
- **UI**：修改 JEI 搜索栏的输入匹配逻辑，无新增 UI。
- **资源**：无额外资源包需求。
- **脚本**：无 KubeJS 交互。
- **加载顺序假设**：标准加载顺序即可；实际通过注入 JEI 搜索实现。

## 配置意图

`config/jecharacters-client.toml` 选择覆盖常见中文输入习惯的拼音方案：

- `enumKeyboard = "QUANPIN"`：使用全拼作为默认键盘方案，符合大多数玩家习惯。
- 启用多组模糊音：
  - `enableFZh2z = true`：`zh` 与 `z` 互通。
  - `enableFSh2s = true`：`sh` 与 `s` 互通。
  - `enableFCh2c = true`：`ch` 与 `c` 互通。
  - `enableFAng2an = true`、`enableFIng2in = true`、`enableFEng2en = true`：前后鼻音互通。
  - `enableFU2v = true`：`u` 与 `v` 互通。
- `enableChat = true`：进入世界时仍保留欢迎提示，便于玩家确认模组已加载。
- `enableQuote = false`：不禁用 JEI 搜索引号分词，保持默认搜索语法。
- `enableVerbose = false`：关闭调试日志，避免污染日志输出。

`config/jecharacters-extra.json` 当前为空（`removals`、`contains`、`equals`、`suffix`、`regExp` 的 `additional` 列表均为空），未添加自定义拼音匹配规则，后续若有 KubeJS 自定义中文物品需要特殊匹配，可在此扩展。

## 兼容性与性能

- **客户端/服务端范围**：CLIENT ONLY。服务端无需安装。
- **存档影响**：无。
- **已知不兼容**：与除 JEI 外的其他配方查看器（REI / EMI）的搜索集成未在当前包中验证。
- **资源成本**：仅在玩家输入搜索词时进行拼音匹配计算，对 JEI 索引加载时间影响可忽略。
- **缓解措施**：模糊音组合已覆盖主流习惯，若个别玩家需要更严格或更宽松的匹配，可在客户端配置中调整。

## 风险与开放问题

1. JEI 版本升级后，其搜索 API 变更可能导致拼音搜索注入失效，需在 JEI 更新后回归验证。
