## 设计作用

Jade Addons 是 Jade 信息提示器的扩展模组，为 Create、Tinkers' Construct、Lootr 等内容模组提供额外的方块/实体提示信息。它在 ProjectNautic 整合包中扮演**前向兼容钩子**的角色：当前目标内容模组大多尚未安装，但预先加入 Jade Addons 可确保未来添加这些模组时，Jade 的提示信息能够自动扩展，无需额外发版更新。

该模组属于 `integration` 分类：它本身不添加独立游戏内容，而是增强 Jade 与其他内容模组之间的协同工作能力。

## 玩家可见行为

- **可发现性**：当前阶段玩家不会看到 Jade Addons 的明显效果，因为目标内容模组（Create、Tinkers' 等）尚未安装。
- **交互**：无直接交互界面。效果通过 Jade 的提示框在未来内容模组安装后自动体现。
- **进度/奖励**：无直接进度关联。
- **摩擦**：无额外摩擦。未安装目标模组时，Jade Addons 几乎无运行时开销。
- **失败状态**：若目标模组版本不兼容，可能导致提示信息缺失或控制台警告。

## 集成关系

- **依赖**：Minecraft 1.21.1、NeoForge、Jade `>=15.10`（required, BOTH）
- **冲突**：当前批次中无已知硬冲突。
- **协作**：

  - **Create / Tinkers' Construct / Lootr 等**：未来可能添加的内容模组，Jade Addons 将为它们的方块/实体提供更详细的提示。例如：Create 的机械网络应力、Tinkers' 的冶炼炉状态、Lootr 宝箱的共享状态等
- **配方/标签**：无直接参与
- **任务**：无
- **世界生成**：无
- **UI**：通过 Jade 的提示框展示扩展信息
- **资源**：无
- **脚本**：无 KubeJS 脚本交互需求
- **加载顺序**：需在 Jade 之后加载

## 配置意图

Jade Addons 本身未提供独立的配置文件。其行为由 Jade 的主配置 `config/jade/plugins.json` 统一管理。当前 `plugins.json` 已包含 `jadeaddons` 条目（在第二批模组加入后由扫描器记录），表示 Jade Addons 的插件已注册，等待目标模组就位后生效。

当前未对 Jade Addons 的各子插件进行单独启用/禁用调整，保留默认全部启用。未来安装 Create、Tinkers' 等模组后，若某些提示信息过于冗余，可再在 `plugins.json` 中精细化管理。

在第二批模组加入后，扫描器已记录 `config/jade/plugins.json` 中新增了 `jadeaddons` 条目，确认插件注册成功。

## 风险与开放问题

1. **前向兼容的空窗期**：当前 Jade Addons 已安装但目标模组缺失，玩家可能困惑于其存在的意义。需在文档中说明这是为未来内容模组预留的扩展。
2. **版本同步风险**：Jade Addons 与 Jade 主版本强绑定，更新 Jade 时必须同步更新 Jade Addons。
3. **提示信息过载**：未来多个目标模组同时安装时，Jade 提示框可能变得冗长，需提前规划信息优先级。
