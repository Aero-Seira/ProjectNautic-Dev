## 设计作用

该资源包是 ProjectNautic 的中文本地化基线，为大量没有官方中文翻译的模组提供统一、社区维护的简体中文翻译。它支撑整合包的**可访问性**与**本土化体验**支柱，让玩家无需自行寻找汉化补丁即可理解物品、界面、提示与任务文本。

作为纯资源包，它不引入新机制，但直接影响玩家对整合包内容的第一认知；翻译质量与一致性会反馈到 JEI、Jade、进度、UI 等所有依赖文本的系统上。

## 玩家可见行为

- **可发现性**：玩家在首次进入游戏时，菜单与物品名称即显示中文；若资源包未生效，大量模组内容会回退为英文或出现混合语言。
- **交互**：玩家可在"资源包"菜单中手动启用/禁用或调整加载顺序；通常默认已启用。
- **进度/奖励**：无直接进度关联，但进度、任务、JEI 提示的中文可读性依赖此包。
- **摩擦/失败**：资源包版本与游戏版本不匹配时，可能无法加载或导致部分翻译缺失；与 I18nUpdateMod 等自动汉化更新模组同时存在时，需确认优先级避免覆盖冲突。

## 集成关系

- **依赖**：Minecraft 1.21.1 资源包格式；依赖模组本身已提供英文 key。
- **冲突**：与另一个同为中文翻译的资源包同时启用时，后加载的包会覆盖前者；与 I18nUpdateMod 的动态下载翻译也可能产生 key 级覆盖。
- **协作**：与 `i18nupdatemod`（自动汉化更新模组）共同构成整合包的双保险汉化方案；与 JEI、Jade、Better Advancements、任务书等所有显示文本的模组协作。
- **配方/标签**：无。
- **任务**：无。
- **世界生成**：无。
- **UI**：覆盖所有依赖 lang 文件的 GUI 文本、提示与物品名称。
- **资源**：该包本身即为资源文件集合，无额外资源需求。
- **脚本**：无 KubeJS 交互。
- **加载顺序假设**：作为语言资源包，通常应置于较高优先级以覆盖模组内置英文；若 I18nUpdateMod 提供更新的在线翻译，需通过加载顺序决定最终显示。

## 配置意图

该资源包无独立配置文件。其设计意图完全由资源包本身的内容与加载顺序体现：在默认启用状态下，为整合包内所有支持的中文 key 提供社区翻译，减少玩家在 JEI、提示、界面中的语言障碍。

## 兼容性与性能

- **客户端/服务端范围**：CLIENT ONLY。资源包仅影响本地客户端渲染的文本。
- **存档影响**：无。
- **已知不兼容**：与不同 Minecraft 版本或不同资源包格式版本的语言包不兼容；同时启用多个中文语言包会导致 key 覆盖混乱。
- **资源成本**：仅增加少量内存占用（lang 文件缓存）与包加载时间，对帧率无直接影响。
- **缓解措施**：保持资源包版本与整合包 Minecraft 版本一致；默认仅启用一个中文语言包；若使用 I18nUpdateMod，监控其在线翻译与本地的覆盖关系。

## 风险与开放问题

1. **翻译滞后风险**：社区语言包更新节奏可能落后于模组版本，新增物品或机制可能出现未翻译或旧译名不一致的情况。
2. **与 I18nUpdateMod 的优先级漂移**：两个汉化来源同时存在时，玩家本地行为可能与开发者预期不同，需在说明文档中明确推荐加载顺序。
