## 设计作用

Ferrite Core 是 ProjectNautic 整合包的内存优化模组，通过压缩方块状态表示、去重模型数据、优化邻居查找表等手段，显著降低客户端与服务端的内存占用。它与 ModernFix、Lithium、Sodium 共同构成内存-CPU-渲染三位一体的性能优化体系，支撑整合包**性能优先**与**长线运营优先**的设计支柱：内存占用的降低意味着服务器可以在相同硬件上承载更多玩家或更大视距，客户端也能在低端设备上稳定运行。

Ferrite Core 标记为 `BOTH`，服务端与客户端均需安装以获得最佳效果。

## 配置意图

`config/ferritecore-mixin.toml` 为默认生成配置，未做覆盖。当前启用项反映上游推荐的稳定优化组合：

- `cacheMultipartPredicates = true`：缓存 multipart 模型谓词，减少重复计算。
- `multipartDeduplication = true`：对使用相同 multipart 模型的方块状态去重实例。
- `blockstateCacheDeduplication = true`：去重方块状态缓存数据（碰撞与渲染形状）。
- `dataComponentPatch = true`：减少空数据组件补丁的内存开销。
- `modelResourceLocations = true`：避免创建新的模型资源位置字符串。
- `modelSides = true`：对简单模型使用更小的数据结构。
- `replaceNeighborLookup = true` / `replacePropertyMap = true`：用更紧凑的邻居查找表与属性映射替代原版实现。
- `bakedQuadDeduplication = true`：去重烘焙四边形的顶点数据。

以下两项保持关闭（默认）：
- `compactFastMap = false`：更紧凑但略慢的方块状态表示，当前未启用。
- `useSmallThreadingDetector = false`：因极罕见的崩溃风险而默认关闭。

这些默认值在安全与收益之间取得了平衡，符合第二批扩展阶段"稳定优先"的策略。

## 兼容性与性能

- **客户端/服务端范围**：BOTH。两侧均需安装以获得完整收益。
- **存档影响**：无直接存档格式影响。优化对象是运行时内存数据结构。
- **已知不兼容**：极少数直接访问原版邻居表或属性映射的模组可能不兼容；当前批次中未发现。
- **资源成本**：无额外 CPU 或显存开销；主要收益为内存占用下降。
- **缓解措施**：保持默认配置；若遇到模型或方块状态相关崩溃，可尝试关闭 `replaceNeighborLookup` 或 `replacePropertyMap`。

## 风险与开放问题

1. **与未知模组的兼容性**：Ferrite Core 修改的是底层数据结构，未来添加大量自定义方块状态的内容模组时需谨慎验证。
2. **调试复杂度**：内存优化问题往往难以定位；建议在加入大型内容模组前后分别测量内存基线。
