## 设计作用

CullLeaves 是 ProjectNautic 整合包的客户端树叶渲染优化模组。它通过剔除被其他树叶方块遮挡的内部树叶面，显著减少茂密森林、丛林、深色橡树林等场景下的 GPU 渲染负载。该模组与 Sodium（区块渲染优化）协同工作，进一步巩固整合包**性能优先**的设计支柱，同时保留原版树叶的视觉风格（不降低树叶质量设置）。

CullLeaves 仅作用于 CLIENT 侧，不影响服务端逻辑或世界生成。

## 玩家可见行为

- **可发现性**：玩家通常不会意识到模组存在，但会在茂密树林中观察到更高的帧率。
- **交互**：无直接交互界面。可通过 MidnightLib 的配置界面切换开关。
- **进度/奖励**：无直接进度关联。
- **摩擦**：无额外摩擦。默认启用树叶与树根剔除。
- **失败状态**：若与某些依赖树叶内部面渲染的视觉效果模组冲突，可能导致树叶出现空洞或闪烁。

## 集成关系

- **依赖**：Minecraft `[1.21,1.21.1]`、NeoForge `>=20.3`、MidnightLib `>=1.0.0`（required, CLIENT）

- **协作**：与 Sodium 共同优化树叶渲染；MidnightLib 提供配置界面支持。

- **脚本**：无 KubeJS 脚本交互需求

## 配置意图

`config/cullleaves.json`：

```json
{
  "enabled": true,
  "cullRoots": true
}
```

- **`enabled: true`**：启用树叶剔除，这是模组的核心功能。
- **`cullRoots: true`**：额外启用树根方块（mangrove roots）的剔除，进一步优化红树林等生物群落的渲染。

该配置保持模组默认行为，未做额外调整。与 Sodium 的 `leaves_quality: DEFAULT` 配置保持一致，确保在获得性能提升的同时不牺牲树叶视觉质量。

## 兼容性与性能

- **客户端/服务端范围**：CLIENT ONLY。服务端无需安装。
- **存档影响**：无。CullLeaves 不修改存档数据或世界生成。
- **已知不兼容**：当前批次中无已知冲突。理论上与某些自定义树叶渲染管线的光影包或资源包可能产生视觉差异。
- **资源成本**：几乎无额外开销；主要收益为茂密树林场景下的 GPU 负载下降。
- **缓解措施**：若玩家遇到树叶空洞等视觉问题，可通过 MidnightLib 配置界面临时禁用。

## 风险与开放问题

1. **视觉一致性**：不同资源包对树叶内部面的依赖程度不同，未来若引入自定义树叶纹理资源包需验证剔除效果。
2. **MidnightLib 版本绑定**：CullLeaves 依赖 MidnightLib 的配置系统，两者需同步更新。
