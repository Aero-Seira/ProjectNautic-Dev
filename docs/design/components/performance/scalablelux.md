## 设计作用

ScalableLux 是 ProjectNautic 整合包的光照引擎重写模组，旨在修复原版 Minecraft 光照更新性能问题与光照错误。它通过替换原版的区块光照计算实现，降低大规模建筑、世界生成与区块加载时的光照更新开销。该模组支撑整合包**性能优先**的设计支柱，特别对服务端长期运行中频繁发生的光照抖动与"光照漏洞"问题有明显改善。

ScalableLux 标记为 `BOTH`，服务端与客户端均可受益；服务端收益尤为显著（降低光照更新导致的 MSPT 峰值）。

## 玩家可见行为

- **可发现性**：玩家不会看到模组 UI，但会体验到：
  - 大规模放置/破坏方块后光照更新更迅速
  - 世界生成或预生成过程中光照错误（如随机黑斑）减少
  - 高移动速度下区块光照加载更连贯
- **交互**：无直接交互界面。
- **进度/奖励**：无直接进度关联。
- **摩擦**：无额外摩擦。安装后自动接管光照引擎。
- **失败状态**：alpha 版本可能与依赖原版光照行为的模组产生冲突，导致光照异常或崩溃。

## 配置意图

未检测到独立的 ScalableLux 配置文件（在 `config/` 目录下未发现 `scalablelux.*`）。模组使用内置默认行为运行，未做覆盖。这符合第二批扩展阶段"最小化 alpha 版本配置风险"的策略：在光照引擎这类底层系统上，优先采用上游默认实现，待稳定后再根据实测数据调整。

与 ScalableLux 相关的唯一可见配置项是 Sodium Extra 的 `light_updates: true`（`config/sodium-extra-options.json`），该选项保持光照更新启用，与 ScalableLux 形成协同：Sodium Extra 决定玩家是否看到光照动画/更新效果，ScalableLux 决定这些更新背后的计算效率。两者配置方向一致，未做冲突性覆盖。

## 兼容性与性能

- **客户端/服务端范围**：BOTH。服务端强烈建议安装，以显著降低光照更新相关的 MSPT。
- **存档影响**：理论上不改变光照数据存储格式；alpha 版本建议在重大更新前备份存档。
- **已知不兼容**：当前批次中无已知冲突。应避免同时安装 Phosphor、Starlight 等同类光照引擎模组。
- **资源成本**：光照计算从原版的同步阻塞模式改为更高效的实现，CPU 占用降低；内存开销变化取决于具体场景。在大型建筑或 Chunky 预生成时，光照更新阶段的 MSPT 峰值通常明显下降。
- **缓解措施**：保持默认配置；若出现光照异常，优先检查是否与自定义维度或照明模组冲突。建议在重大建筑活动前后对比 spark 的 MSPT 报告以量化收益。

## 风险与开放问题

1. **Alpha 版本稳定性**：`0.3.0-alpha.0.6` 属于早期版本，光照引擎是 Minecraft 的核心系统之一，潜在 bug 可能导致世界光照数据损坏。
2. **与光影包/渲染模组的兼容性**：Iris 等光影模组可能有自己的光照处理路径，需验证 ScalableLux 是否与之协同或竞争。
3. **服务端与客户端版本一致性**：作为 `BOTH` 侧模组，两侧版本必须严格一致。
