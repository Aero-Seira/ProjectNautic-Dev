## 设计作用

Smooth Scrolling 为 ProjectNautic 提供快捷栏、聊天栏、创造模式物品栏及各类条目列表的平滑滚动，直接服务于"UI 愉悦感"设计支柱。它将原版瞬跳的滚动替换为带惯性的平滑滚动，使玩家在切换物品、浏览聊天或翻页时感觉更流畅。

该模组与 Smooth GUI 共同构成 ProjectNautic 的界面动效层：Smooth Scrolling 负责列表与快捷栏滚动，Smooth GUI 负责屏幕整体出现/消失动画。

## 玩家可见行为

- **可发现性**：玩家滚动鼠标滚轮切换快捷栏、浏览聊天历史、翻创造模式物品栏或各类配置列表时，内容会以平滑动画方式移动。
- **交互**：无需主动操作；玩家可在配置文件中调整各区域的平滑度与速度。
- **进度/奖励**：无内置进度；平滑滚动作为日常操作的视觉润色。
- **摩擦/失败**：平滑度过高可能让玩家感觉操作有延迟或"粘滞"；某些模组的自定义列表可能不被支持。

## 配置意图

`config/smoothscrolling-client.toml` 当前配置对各滚动区域做了差异化平滑度设置：

- **Hotbar**：`smoothness=0.5`，`rollover=true`。快捷栏滚动平滑度调至中等，与聊天/列表对齐，提升滚动物理感；同时启用滚轮越过首尾时循环滚动，提升操作效率。
- **Chat**：`smoothness=0.5`，`openingSmoothness=0.5`。聊天栏滚动与打开动画均采用中等平滑度，兼顾流畅感与可读性。
- **Creative Screen**：`smoothness=0.5`。创造模式物品栏滚动采用中等平滑度，避免翻页过快导致找不到物品。
- **Entry List**：`smoothness=0.5`，`speed=30.0`。条目列表（如配置界面、模组列表）滚动中等平滑，自动速度为 30 px，保持适中响应。
- **Misc**：`enableMaskDebug=false`，关闭遮罩调试可视化。

整体配置意图是"快捷栏求快、列表求稳"：对需要快速反应的快捷栏降低平滑度，对需要精确浏览的聊天/创造/列表提高平滑度。

## 风险与开放问题

1. 与 Modern UI 的视图系统或自定义滚动组件叠加时，可能出现平滑度重复计算或滚动错位。
2. 部分模组自定义的条目列表可能无法被 Smooth Scrolling 识别，导致体验不一致。
