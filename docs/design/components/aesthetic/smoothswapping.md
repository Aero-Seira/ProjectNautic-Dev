## 设计作用

Smooth Swapping 为 ProjectNautic 提供物品在物品栏格子之间移动时的平滑位移动画，直接服务于"UI 愉悦感"设计支柱。它用流畅的动画替代原版瞬间切换，使整理背包、合成、使用漏斗等操作在视觉上更连贯。

该模组与 Tiny Item Animations、Inventory Particles、Extra Sounds 共同构成 ProjectNautic 的物品栏体验层：Smooth Swapping 负责位移动画，Tiny Item Animations 负责缩放动画，Inventory Particles 负责粒子轨迹，Extra Sounds 负责音效。

## 玩家可见行为

- **可发现性**：玩家移动物品栏中的物品时，物品图标会从原位置平滑飞移到目标位置，而不是瞬间跳转。
- **交互**：无需主动操作；动画在物品位置变化时自动触发。
- **进度/奖励**：无内置进度；动画作为物品操作的视觉奖励。
- **摩擦/失败**：动画速度过慢可能让玩家感觉操作延迟；与某些模组自定义物品栏布局可能产生错位。

## 配置意图

`config/smoothswapping.json` 当前保留了模组默认值，意图是让移动动画自然流畅：

- **`toggle_mod=true`**：启用平滑转移功能。
- **`animation_speed=100`**：默认动画速度，控制物品从原位置飞移到目标位置所需时间。
- **`curve_points=[[0.1875, 0.875]]`**：默认贝塞尔曲线控制点，决定物品移动的加减速曲线，使动画在开始和结束时有自然的缓动效果。

整合包未对参数做自定义，说明当前设计意图是接受模组作者预设的动画节奏，作为对原版物品栏操作的轻量润色。

## 兼容性与性能

- **客户端/服务端范围**：仅 **CLIENT**。
- **存档影响**：无。
- **已知不兼容**：无。
- **资源成本**：纯 CLIENT 端 UI 动画，开销极低；大量物品同时移动时 GPU 需绘制多个飞移中的图标。
- **缓解措施**：使用默认动画速度与曲线，避免极端慢速导致操作感知延迟。

## 风险与开放问题

1. 与 Inventory Particles 叠加时，粒子可能从动画起点而非终点生成，导致视觉不连贯；需实机测试。
2. 与某些自定义物品栏界面（如 JEI 书签栏、模组专用容器）可能产生渲染错位。
