## 设计作用

AmbientSounds 是 ProjectNautic 的环境声景核心，通过程序化混音为不同维度、生物群系与天气状态生成连续的环境音效。它直接支撑"沉浸感"支柱，将静态的方块世界转化为具有风声、鸟鸣、海浪、洞穴回响的动态声场。

在整合包中，AmbientSounds 与 Ambiance、Extra Sounds 形成分层音频：AmbientSounds 负责宏观环境底噪，Ambiance 负责特定对象的氛围细节，Extra Sounds 负责 UI 与交互反馈。三者音量均经过控制，避免相互掩蔽。

## 玩家可见行为

- **可发现性**：玩家进入不同生物群系、维度、天气或洞穴时，背景环境音会平滑过渡；海边有海浪与海鸥，森林有鸟鸣与虫鸣，沙漠有风声与蝉鸣，洞穴有滴水与低沉回响。
- **交互**：无需主动交互；玩家可通过模组配置菜单或配置文件调节主音量、各区域音量以及淡入淡出速度。
- **进度/奖励**：无内置进度；环境音效作为探索不同生态的感官奖励。
- **摩擦/失败**：音效引擎扫描世界方块时可能带来轻微 CLIENT 端负载；若与同样添加环境音的模组叠加，可能出现声音重复或音量冲突。

## 配置意图

`config/ambientsounds-client.json` 当前保留了默认的全区域声景配置，仅在顶层参数上保持平衡：

- **`general`**：`volume=1` 保持默认主音量；`useSoundMasterSource=false` 不使用主音源混合，便于玩家在游戏音量菜单中独立控制；`scanStepAmount=100` 保留默认扫描步长；`playSoundWithOffset=true` 启用随机偏移播放，避免环境音循环单调。
- **`dimensions`**：末地、下界、洞穴、地表、虚空等维度均启用（值为 1），确保跨维度旅行时声景连续。
- **`regions`**：几乎所有区域声景均启用，包括海滩、森林、丛林、沙漠、海洋、沼泽、苍白之园、风暴、洞穴等。每个区域下的子音效（如 `beach/seagull`、`cave/cave-deep`、`ocean/whale`）也保持开启，意图是最大化生物群系辨识度。
- **`categories`**：动物、洞穴、悬疑、天气、风等分类全部启用，作为区域音效的混合基础。
- **`fade-volume` 与 `fade-pitch`**：均设为 `0.005`，保证环境音切换平滑自然，不出现突兀切歌感。
- **`silent-dimensions`**：为空列表，未强制静音任何维度。

## 兼容性与性能

- **客户端/服务端范围**：仅 **CLIENT**。
- **存档影响**：无。
- **已知不兼容**：无。
- **资源成本**：环境音效为持续播放的 CLIENT 端音频流，内存占用主要来自音频缓冲；`scanStepAmount=100` 在控制扫描开销与定位精度之间取得平衡。
- **缓解措施**：主音量保持默认 1.0，但玩家可在游戏中独立调节；未启用极端密集的扫描配置。
