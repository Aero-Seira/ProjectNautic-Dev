## 设计作用

Extra Sounds 为 ProjectNautic 提供额外的 UI 与交互音效，直接服务于"UI 愉悦感"和"沉浸感"设计支柱。它补充了原版缺失的背包开关、物品拾取/拖拽、快捷栏滚动、聊天输入、药水效果变化等声音反馈，让玩家的日常操作更有质感。

该模组与 AmbientSounds 共同构成 ProjectNautic 的音频体验层：AmbientSounds 负责宏观环境底噪，Extra Sounds 负责 UI 与交互音效，两者音量独立可控。

## 玩家可见行为

- **可发现性**：玩家打开/关闭背包、滚动背包、拾取/拖拽/丢弃物品、切换快捷栏、输入聊天文字、获得/移除药水效果、拉弓、调整中继器、实体消散时都会听到额外音效。
- **交互**：无需主动操作；玩家可通过配置文件或模组菜单调整各音量通道与开关。
- **进度/奖励**：无内置进度；音效作为操作的听觉反馈。
- **摩擦/失败**：过多或过大的 UI 音效可能让玩家感到烦躁；某些玩家可能希望完全关闭部分音效。

## 配置意图

`config/extrasounds.json` 当前配置对音量通道做了明显平衡：

- **主音量**：`master=0.5`，将 Extra Sounds 整体音量减半，避免 UI 音效盖过游戏环境音与音乐。
- **分通道音量**：`inventory=1.0`、`action=1.0`、`chat=1.0`、`effects=1.0`、`hotbar=1.0`、`typing=1.0`、`item_drop=1.0`、`empty_hotbar=1.0`、`enabled_footstep=1.0` 均保持默认 1.0，表示在主音量减半的前提下，各子类别相对平衡。
- **特效音量**：`enabled_effects=0.0`、`enabled_poof=0.0`，明确关闭了实体消散/特效类音效，避免过于花哨或与原版音效冲突。
- **音效开关**：`sounds` 下几乎所有项均为 `true`，包括聊天、背包开关/滚动、物品拾取/拖拽/丢弃/复制/删除、快捷栏滚动、键盘输入/移动/删除/剪切/粘贴、药水效果增减、拉弓、中继器调整、实体消散等。意图是全面增强交互反馈，但通过 `master=0.5` 和关闭特效音量保持克制。

## 兼容性与性能

- **客户端/服务端范围**：仅 **CLIENT**。
- **存档影响**：无。
- **已知不兼容**：无。
- **资源成本**：音频为轻量 CLIENT 端开销；大量连续 UI 音效（如快速整理背包）可能短暂增加音频线程负载。
- **缓解措施**：`master=0.5` 降低整体音量与感知强度；`enabled_effects=0.0` 关闭可能重复的特效音效。
