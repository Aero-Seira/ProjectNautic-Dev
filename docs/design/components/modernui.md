## 设计作用

Modern UI 是 ProjectNautic 的界面风格基础模组，直接支撑"UI 一致性"设计支柱。它通过替换 Minecraft 原生的字体渲染管线、重写工具提示绘制逻辑、以及提供现代化的屏幕布局框架，为整合包内所有模组的界面元素建立统一的视觉语言。

在 ProjectNautic 中，Modern UI 的定位不是单纯的"美化模组"，而是视觉体验的基础设施：
- 它确保无论玩家与哪个模组的 GUI 交互，文字清晰度、阴影处理、间距节奏都保持一致。
- 它为整合包后续通过 KubeJS 或数据包扩展自定义界面时提供了可预期的渲染基线。
- 它与 Sodium 的渲染优化互不冲突，但需确保其底层图形 API 调用不会与 Iris 的着色器后处理产生渲染顺序问题。

## 玩家可见行为

- **可发现性**：玩家从主菜单、暂停菜单、物品栏、工具提示等全局界面中直接感受到字体和布局的变化；Modern UI 本身没有独立的"设置入口"，其效果是无处不在的。
- **交互**：
  - 工具提示（Tooltip）渲染更平滑，支持更丰富的文本格式化与背景模糊效果。
  - 字体渲染采用 SDF（Signed Distance Field）或类似技术，在高分辨率 / GUI 缩放下边缘更清晰。
  - 部分屏幕（如配置界面）可能采用 Modern UI 提供的现代化布局替代原生布局。
- **进度**：无内置进度系统；安装即全局生效。
- **奖励**：更舒适的长时间阅读体验；界面视觉层级更清晰，减少认知负荷。
- **摩擦**：
  - 习惯原版像素字体的玩家可能需要适应期。
  - 若配置不当，某些模组的自定义 GUI 可能与 Modern UI 的渲染假设冲突，导致文字截断或布局错位。
- **失败状态**：
  - 与 TipTheScales 同时使用时可能出现 GUI 缩放冲突或渲染异常。
  - 极端自定义字体配置下，若指定字体文件缺失，可能回退到系统默认字体，破坏视觉一致性。

## 集成关系

- **依赖**：
  - NeoForge `>= 21.0`（required, BOTH）
  - Minecraft `[1.21, 1.21.2)`（required, BOTH）
- **冲突/警告**：
  - 不建议与 **TipTheScales** 同时使用（discouraged）。原因：两者均涉及 GUI 缩放和屏幕渲染逻辑，同时存在时可能导致不可预期的布局错位或缩放计算冲突。ProjectNautic 模组清单中应避免引入 TipTheScales。
- **协作**：
  - 与 Sodium / Iris 在 CLIENT 端共存，需验证渲染管线互不干扰。
  - 为 KubeJS 脚本系统中可能创建的自定义屏幕提供底层渲染支持。
- **配方/标签**：无直接参与。
- **任务**：无内置任务系统。
- **世界生成**：无影响。
- **UI**：全局替换字体渲染器和工具提示绘制器；可能替换部分配置屏幕的布局框架。
- **资源**：依赖字体资源文件（可通过配置指定自定义字体）；默认使用内置或系统字体。
- **脚本**：无直接 KubeJS API 暴露，但脚本创建的自定义 GUI 将自动受益于 Modern UI 的渲染改进。
- **加载顺序假设**：作为 BOTH 端模组，在 NeoForge 主加载周期中初始化；其 CLIENT 端渲染器注册需在 Minecraft 字体管理器初始化之后完成。

## 配置意图

Modern UI 的配置文件体系庞大（client.toml 达 218 行），设计文档不逐条记录默认值，仅聚焦于表达深思熟虑的设计选择：

### `config/ModernUI/client.toml`

该文件按顶级键划分为 `screen`、`tooltip`、`general`、`view`、`font` 五个域。ProjectNautic 的配置意图如下：

- **`screen`**：控制屏幕布局与动画行为。整合包保留默认设置，意图是让 Modern UI 的现代化屏幕框架自然接管，不强制关闭动画（避免过于"生硬"），也不开启过度复杂的过渡效果（保持性能优先）。
- **`tooltip`**：工具提示是玩家最频繁接触的界面元素。此域的配置决定了工具提示背景样式（如是否启用模糊、圆角、阴影深度）。整合包选择启用增强型工具提示渲染，以直接提升"UI 一致性"——确保所有模组（包括原版和第三方）的提示框在视觉风格上统一。
- **`general`**：包含全局开关和性能相关选项。整合包保留默认的自动优化设置，不启用实验性渲染路径，以维持稳定性（尤其考虑到 Iris 尚处于 beta 阶段，渲染管线不宜叠加过多实验特性）。
- **`view`**：涉及视图层级和滚动行为。保留默认配置，意图是让 Modern UI 的视图系统在不破坏原版操作习惯的前提下渐进式改进。
- **`font`**：字体配置是 Modern UI 的核心差异化能力。整合包保留默认字体选择逻辑，不强制绑定某一特定字体文件，原因是：
  - 强制字体可能导致中文字符集覆盖不足或授权问题。
  - 默认逻辑通常能自动选择系统上可用的合适字体（如 Windows 上的 Segoe UI、macOS 上的 SF Pro、Linux 上的 Noto Sans 等），兼顾跨平台一致性。
  - 若未来整合包定义了专属品牌字体，可在此域统一指定，届时需补充字体资源包和授权文件。

### `config/ModernUI/common.toml`

- **`developer`**：保留关闭状态。生产环境中不启用开发者调试绘制（如布局边界框、性能计数器），避免向终端玩家暴露内部信息。
- **`autoShutdown`**：保留默认策略。该功能与整合包服务端管理逻辑无关，由外部面板或脚本统一控制服务器生命周期。

### `config/ModernUI/text.toml`

- **`text`**：控制文本渲染的高级特性（如双向文本、排版优化、行高调整）。整合包保留默认配置，意图是让国际化文本（包括中文）获得正确的排版处理，同时不引入过于激进的行距或字距调整，保持与原版 Minecraft 的排版节奏兼容。

### `config/ModernUI/bootstrap.properties`

- 该文件当前为空（0 B）。保留空文件作为占位，以便未来在启动早期阶段需要注入 JVM 或渲染器参数时，无需更改模组文件结构。

## 兼容性与性能

- **客户端/服务端范围**：**BOTH**。虽然 Modern UI 的核心价值体现在 CLIENT 端的视觉改进，但其作为 BOTH 端模组存在，服务端侧主要处理配置同步和可能的网络协议支持。服务端运行时的额外开销极小。
- **存档影响**：无直接存档格式影响；所有配置存储于配置文件，不写入世界 NBT 或玩家数据。
- **已知不兼容**：
  - **TipTheScales**：官方明确标记为 discouraged，同时使用时 GUI 缩放计算可能冲突，导致界面元素尺寸异常或位置偏移。
  - 部分重度自定义 GUI 的模组（尤其是直接操作 `FontRenderer` 或 `GuiGraphics` 的低级模组）可能需要兼容性补丁。
- **资源成本**：
  - 字体渲染的 SDF 管线在 GPU 上开销极低，但会占用少量显存用于字体图集缓存。
  - 工具提示模糊效果（如启用）在低端 GPU 上可能有轻微性能影响；ProjectNautic 默认配置保持平衡。
- **缓解措施**：
  - 在模组引入审核流程中检查是否有 TipTheScales 或同类 GUI 缩放模组的间接依赖。
  - 在目标硬件配置上进行 CLIENT 端启动和界面遍历测试，确保字体渲染不会成为帧率瓶颈。

## 风险与开放问题

1. **配置文件维护复杂度**：client.toml 达 218 行，且 Modern UI 版本迭代时配置键可能增删或重命名。需要建立版本升级时的配置差异审查流程，避免升级后配置失效回退到默认值。
2. **中文字体渲染质量**：默认字体选择逻辑在不同操作系统上行为不同，Windows 上可能选中宋体或微软雅黑，Linux 上可能选中 Noto Sans CJK。需要在中文环境下进行专门测试，确保字形清晰、字重合适、无错位或截断。
3. **KubeJS 自定义屏幕兼容性**：若未来通过 KubeJS 创建自定义 HUD 或全屏界面，需验证这些界面是否正确继承 Modern UI 的渲染上下文，避免"部分界面现代化、部分界面原版化"的视觉分裂。
4. **BOTH 端服务端开销**：虽然理论上开销极小，但需在多人测试阶段确认服务端不因为 Modern UI 的网络同步逻辑产生异常 TPS 波动。
