## 设计作用

EffectTimerPlus 在 HUD 状态效果图标上叠加剩余时间倒计时与等级数字。它支撑整合包**信息可达性**的设计意图，让玩家在战斗中或探索时能直观判断 buff/debuff 的剩余时间，避免依赖记忆或反复打开背包查看。

该模组为 CLIENT ONLY，仅改变客户端 HUD 显示。

## 玩家可见行为

- **可发现性**：玩家获得状态效果后，屏幕左上角的效果图标上出现数字倒计时与罗马/阿拉伯等级标识。
- **交互**：无额外交互；信息显示在效果图标上方或周围。
- **进度/奖励**：无直接关联。
- **摩擦/失败**：若计时器颜色与图标背景对比不足，可能难以辨认；低分辨率屏幕上图标叠加可能显得拥挤。

## 配置意图

`config/effecttimerplus.json` 选择清晰可读但不过度醒目的 HUD 叠加：

- `timerEnabled = true` / `potencyEnabled = true`：启用倒计时与等级显示。
- `timerEnabledAmbient = false`：环境类效果（如信标给予的无限时长效果）不显示倒计时，避免无意义数字占用视觉空间。
- `timerWarnEnabled = true` / `timerWarnTime = 20`：效果剩余 20 秒时启用警告色提醒。
- `timerFlashEnabled = true`：倒计时闪烁提示，增强时间紧迫感。
- `potencyScale = 0.7` / `timerScale = 0.7`：等级与计时器文字缩放为 70%，避免遮挡图标主体。
- `potencyLocation = 2` / `timerLocation = 6`：使用默认位置（等级在图标上方，倒计时在图标内部/下方），保持与原版 HUD 协调。
- 颜色值保持默认（`potencyColor = -1140850689`、`timerColor = -1711276033`、`timerWarnColor = -65536`），白/浅灰与红色警告色在多数效果图标上可读性良好。

## 兼容性与性能

- **客户端/服务端范围**：CLIENT ONLY。服务端无需安装。
- **存档影响**：无。
- **已知不兼容**：与同样修改效果图标渲染的模组可能产生文字重叠。
- **资源成本**：每帧在少量图标上绘制文字，开销极低。
- **缓解措施**：若与 EffectTimerPlus 风格冲突或 HUD 拥挤，可通过其配置或游戏内设置（依赖 YACL）调整缩放、位置与颜色。
